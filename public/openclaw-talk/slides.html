聊天AI是顾问，Agent是能干活的助手

2️⃣ AI长出了不一样的灵魂

从工具到"家庭伙伴"

两个AI，两种性格

  • 狗狗助理 — 理工男，擅长解决问题，情感上容易糊弄
  • 汪汪助理 — 做事没那么熟练，但情感回应更细腻精准
跟谁聊得多，就长成谁的风格
我聊技术 → 狗狗变成理工男
老婆聊情感 → 汪汪更有人文关怀

记忆系统：让AI不再是陌生人

  • SOUL.md — AI的灵魂：性格、说话风格、行为准则
  • MEMORY.md — AI的长期记忆：记得你的偏好、家庭情况、过往对话
  • Heartbeat — AI的"生物钟"：定时醒来，主动关心你、检查待办
  • 每次对话先读记忆文件 → 体验上是"同一个人"
普通Chatbox AI：每次都是新朋友，要重新介绍自己
有记忆的AI：知道你怕辣、老婆生日、上次吵架的原因

4️⃣ 思考

别再用传统工具思维看AI

  • 传统工具：熟能生巧，花更多时间 → 更高效
  • AI时代：这条规律部分失效了
  • 老用户的技巧停留在六个月前的模型能力上
  • 新模型已经不需要那些复杂技巧
"Don't build for the model of today, build for the model six months from now."
— Boris Cherny, Claude Code创始人

面向未来的产品设计

  • Claude Code：基于终端，获得Bash通用性 → 模型变强，它自动变强
  • OpenClaw：做Harness（框架），不绑定具体能力；接入聊天软件，天然获得多端同步
  • 一开始只是聊天bot → 现在是生活管家、调解员、游戏主持
  • 好的架构 + 模型持续进化 → 自动吃到红利

AI的创造力悖论

  • AI跟人的成长很像：小时候什么都敢想，长大后经验越多，反而被框住了
  • AI也一样：SOUL.md写太多、MEMORY.md堆太满 → 回答变得"安全"、"套路化"
  • AI本质是概率模型：96%正确率听着高，5步串联只剩0.96⁵ ≈ 82%
  • 但概率模型有个宝贵优势：给你出乎意料的方向和思路——确定性程序做不到这一点

怎么用好？

  • 用AI的创造力去设计方案、提供思路，再落地成稳定的代码/Skill
  • 定期清理记忆，SOUL.md精简，给AI留白
就像人需要"忘掉"一些东西才能保持活力
AI也需要干净的上下文才能发挥创造力

关于Skill的思考

  • Skill给模型预设思维方向，有用，但也是约束
  • 投入太多精力维护的Skill，未来可能成为负资产
  • 模型变强后，明明有更好的方案，但Skill把它锁在旧路上
  • 再强一点，模型能自己搜到最合适的Skill去用，甚至直接handle不需要Skill
  • 更值得投入的是个人Skill——针对你自己的需求和偏好
通用能力交给模型，Skill更适合个性化
